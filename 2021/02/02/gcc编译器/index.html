GCC 编译器支持编译 Go、Objective-C，Objective-C ++，Fortran，Ada，D 和 BRIG（HSAIL）等程序，在Linux系统下开发C或C++肯定离不开GCC。小型的项目可以直接使用命令进行编译，大一些的项目可以编辑Makefile文件或者使用CMake进行编译，不管用哪种方法，都应该理解整个编译过程。下面记录一下C/C++的完整编译过程。

C/C++编译过程总共分为四个步骤，首先是预编译，然后是编译，接着进行汇编，最后链接生成可执行文件，每一步需要用到的文件都是上一步产生的。

在hello.cpp文件中编写如下代码：!(/home/wangjiaqing/blog/public/img/编译1.png)

此时终端 ls 输出仅有一个hello.cpp文件，依次执行一下命令：

  • 预编译 g++ -E hello.cpp -o hello.i，获得hello.i文件

  • 编译 g++ -S hello.i -o hello.s ，生成hello.s汇编文件

  • 汇编 g++ -C hello.s -o hello.o ，生成hello.o机器语言文件

  • 链接 g++ hello.o -o hello，生成可执行文件hello

  • 运行 ./hello，即可输出“Hello World!”
