1.二分查找原理

二分查找算法又叫折半查找，是常用的查找算法之一,思想简单但是实现起来非常考验细节。二分查找对查找数据有两个要求，一是数列有序，二是数列使用顺序存储结构。下面用一个简单的例子说明二分查找的思想：

0 1 2 3 4 5 6 7 8
4 25 34 56 60 78 80 88 97

以上有一个顺序列表，包含9个元素，现在需要在该列表中查找目标target = 80, 返回它的下标。简单的查找算法需要将所有数比较一遍，二分查找思想是, 首先用目标值与列表中间位置的元素进行比较，中间元素的下标是 (8 + 0) / 2 = 4, 对应的值为60, 明显 80 > 60, 所以就是用与下标从5 - 8的元素进行比较。每次比较都从中间开始, 直到找到与目标值相等的元素，返回它的下标。上述例子完整查找过程如下：

  • 中间元素下标为**(8 + 0) / 2 = 4**, 对应元素值是60, 小于80, 因此剩下查找空间为： 78, 80, 88, 97
  • 此时中间元素下标为**(8 + 5) / 2 = 6**, 对应元素值80, 与目标值相同，查找成功。

3.基本的二分查找(查找一个数)

最简单的二分查找就是查找一个数，和开头的例子是一样的，如果找到这个数，返回其索引，否则返回-1。

此处需要注意几个问题：

(1)为什么while循环的条件中是 <=, 而不是 < ?

这是因为在初始化right时赋值是 nums.length - 1, 即将最后一个元素赋值给right,此时的查找空间是**[left, right]。如果赋值给right的是nums.length**, 则查找空间应该是**[left, right)**。当然如果在查找区间内找到就应该终止：

但是有可能找不到, 就需要终止while循环，返回-1。如果查找区间不为空，就应该一直查找，所以while循环终止时，应该是查找区间为空。

while (left <= right)终止条件是left = right + 1，表示成区间就是[right + 1, right]，显然是空区间，应当终止；

while (left < right)终止条件是left = right，表示成区间就是[right, right]，显然此时查找空间并不为空，查找不应该结束。
